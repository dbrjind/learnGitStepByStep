1. How to set up Git (local machine)

What to do

Install Git:

macOS: brew install git (or download from git-scm.com)

Windows: download Git for Windows and install (includes Git Bash)

Linux: e.g. sudo apt install git or sudo dnf install git

Configure your identity (one-time):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"


(Optional) Check config:

git config --global --list


(Recommended) Set your default branch name:

git config --global init.defaultBranch main


(Optional) Set up SSH keys for easier authentication:

# generate
ssh-keygen -t ed25519 -C "you@example.com"
# start ssh agent and add key (macOS/Linux)
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
# then copy the public key and add to GitHub: ~/.ssh/id_ed25519.pub


Tips

Use SSH for passwordless pushes (after adding the public key to GitHub).

Keep your user email the same as the one on GitHub to link commits to your account.

3. How to clone the created repo locally

Command

# HTTPS
git clone https://github.com/username/my-diary.git

# SSH (recommended if you added an SSH key)
git clone git@github.com:username/my-diary.git


After cloning

cd my-diary
git remote -v   # shows origin URL


Tip
If you started locally and want to connect to a GitHub repo:

git remote add origin git@github.com:username/my-diary.git
git push -u origin main   # set upstream branch

4. How to add a new file to the repo

Create a file

# create README or diary entry
echo "# My Learning Diary" > README.md
mkdir diary
echo "2025-12-13: Learned Git basics" > diary/2025-12-13.md


Or use your code editor to create files.

Check status

git status


Stage file(s)

git add README.md
git add diary/2025-12-13.md

# stage everything (careful)
git add .

    5. How to commit changes

Create a descriptive commit

git commit -m "Add daily diary entry: Git & GitHub basics"


Good commit message format (short + optional body)

Short summary (50 chars or less)

Longer description explaining what and why (optional).


Useful commands

git log --oneline    # compact commit history
git show HEAD        # see latest commit diff
git diff             # show unstaged changes


Tips

Commit often with small focused changes.

Write clear messages: what changed and why.

6. How to push changes to GitHub

Push to remote

# first push and set upstream (if branch not yet tracked)
git push -u origin main

# subsequent pushes
git push


If push is rejected

git pull --rebase origin main  # get upstream changes and reapply local commits
git push


Verify

git remote -v
git branch -vv


Tips

Always git pull (or git fetch + git merge/rebase) before pushing if others change the repo.

Use branches for new features: git checkout -b feature/diary-format.
